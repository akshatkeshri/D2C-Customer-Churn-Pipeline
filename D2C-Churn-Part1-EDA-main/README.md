Part 1: Exploratory Data Analysis (EDA) - D2C Customer Churn

Project Overview
This repository contains Part 1 of a comprehensive capstone project analyzing customer churn for a Direct-to-Consumer (D2C) personal-care brand. The objective of this phase is to clean the raw data, perform exploratory data analysis (EDA), and profile customer behavior to uncover churn-risk patterns.

Repository Contents
* `EDA_Behavior_Profiling.ipynb`: The main Jupyter Notebook containing the data cleaning, visualizations, and business reasoning.
* `requirements.txt`: Python dependencies required to run the notebook.
* `/data`: (To be created locally by the user - see instructions below).

How to Run This Project
To ensure data privacy and maintain clean relative paths, the datasets are not hardcoded. To run this notebook on your local machine:
1. Clone this repository.
2. Install the required packages: `pip install -r requirements.txt`
3. Create a folder named `data` in the root directory of the cloned repository.
4. Place the required dataset files (`customers.csv`, `orders.csv`, `support_tickets.csv`, etc.) into the `data` folder.
5. Run all cells in `EDA_Behavior_Profiling.ipynb`.
